- **Task 3 — Trailer `/ID` from content SHA-256** ✅ (2026-05-01)

- **Post-Task-5 hardening pass** ✅ (2026-05-01) — fifth review-driven round.
  - **Tagged-union invariant locked at the type level (P1)**: `DisplayCommand.Kind` was a public mutable field — any consumer or friend assembly could retag a constructed command, leaving the overlaid payload misinterpreted. Now `_kind` is a private `[FieldOffset(0)]` field with a `public readonly DisplayCommandKind Kind => _kind` getter. Factories assign `_kind` directly; no path exists to retag from outside `DisplayCommand` itself. All `As*` / `Equals` / `GetHashCode` / `ToString` members are now `readonly` so callers passing `in DisplayCommand` skip defensive copies.
  - **Finite-geometry rejection at IR boundary (P2)**: `RectFill`, `TextRun`, `ImageDraw`, and `TransformPush` factories now reject `NaN` / `±∞` for every coordinate, size, and matrix term via `EnsureFinite`. Negative coordinates and sizes remain accepted (PDF's `re` operator allows them; layout uses negatives routinely). Mirrors the round-4 color-component rejection — producer bugs surface at the IR boundary instead of much later in the PDF emitter.
  - **`None` sentinel rejected at `DisplayList.Add` (P2)**: appending a `default(DisplayCommand)` (Kind == None) now throws `ArgumentException`. The sentinel is documented as "uninitialized"; letting it leak into the rendering pipeline forces every downstream consumer to defend against an obviously-invalid command.
  - **Side-table buffer ownership decided (P2)**: `TextRun.GlyphIds` and `TextRun.Advances` now copy on `init` — caller mutations to the source array no longer affect the inserted run. Cost is one short array allocation per shaped run (typically < 1 KB) which preserves the determinism guarantee documented on `DisplayList`. `RasterImage.EncodedBytes` keeps reference semantics (encoded payloads are tens-to-hundreds of KB and the typical path is a shared cache slot — copying every insert would defeat that), with the immutability contract now documented explicitly on the type and on the `EncodedBytes` property.
  - **Boundary validation in `AddTextRun` / `AddImage`**: `AddTextRun` rejects non-positive or non-finite `FontSize` and rejects mismatched `GlyphIds` / `Advances` lengths when either buffer is populated. `AddImage` rejects empty `EncodedBytes` and non-positive pixel dimensions. Contract violations surface at insertion, not at emit.

  - **Unified `RuleSlots` list (Rec 1 + Rec 2)**: replaced `CssPreprocessResult.RulePositions` with a typed `ImmutableArray<CssPreprocessRuleSlot>` ordered by source. Each entry is either `CssAngleSharpRuleSlot` (carries position; the next AngleSharp rule pairs here) or `CssOpaqueAtRuleSlot` (carries name + prelude + position; an at-rule AngleSharp drops, the adapter splices in directly). The preprocessor recognizes the at-keywords AngleSharp drops via a `FrozenSet<string>` allowlist (`"container"`, `"layer"`) and routes them to opaque slots. The adapter's new `AdaptTopLevelRules` walks the slot list — pairing AngleSharp emits sequentially via a cursor, splicing opaque slots inline. This makes the ordinal-drift fix and modern-at-rule capture the same change: modern at-rules become first-class entries in the AST (as `CssAtRule` with `Name = "container"` / `"layer"`, no children, no declarations — opaque pending Task 7's layer cascade and post-v1 container-queries).
  - **Token-aware `!important` recognition (Rec 5)**: the previous `EndsWith("!important")` check would falsely strip the marker from values like `content: "!important"` (the closing-quote bytes precede the literal `!important` text). New `StripTrailingImportant` walks the value text via `CssTokenizer`, skipping strings/comments/parens, and only strips `!important` when found at the value tail outside any token-respecting span. Pinned by an integration test in `CssParserAdapterPreprocessTests`.
  - **Margin-box name allowlist (Rec 6)**: `CssPreprocessor` now validates margin-box names against the 16-entry CSS Paged Media L3 §6.4 list (`top-left-corner`, `top-left`, `top-center`, …, `right-bottom`). Unknown names like `@top-bogus` are silently dropped (treated as malformed CSS) instead of becoming pseudo-margin-boxes. A `[Theory]` covering all 16 names round-trips the recovery; a separate test pins the unknown-name drop.
  - **Nested rule locations now correctly Unknown (Rec 3)**: the previous code threaded the top-level slot list down into `@media`/`@supports`/`@keyframes` child-rule adaptation, so child positions matched random top-level entries. Adapter now has a separate `AdaptNestedRules` path that uses no preprocess data — every nested rule's `Location` is `Unknown`. Recovering nested positions is a future Task 3 follow-up.
  - **Documented limitations (Recs 4 + 7)**: per-property declaration positions inside margin-boxes still resolve to the parent margin-box's location (the tokenizer tracks body-relative positions but converting to absolute requires offset bookkeeping the recovery records don't carry today; documented in `ParseRawDeclarations`). CSS escape sequences in identifiers (`\41 ` = "A") are pinned with a test that locks the current "pass-through verbatim" behavior; generated CSS rarely uses identifier escapes.
  - **Modern value functions still deferred**: `oklch()`/`color-mix()`/`light-dark()` are value-level constructs requiring per-declaration value-text re-parsing — that's plumbed into the typed value tree in Tasks 9–10. AngleSharp's silent corruption of `oklch` to wrong rgba is documented in PROGRESS.md as a Task 9–10 prerequisite.
  - **Adapter API changes**: the preprocess-aware `Adapt(sheet, preprocess, …)` overload now drives the slot-walking path; the simpler `Adapt(sheet, …metadata)` overload still works (delegates with `CssPreprocessResult.Empty` → falls back to AngleSharp-only walk with Unknown positions). Backward compatible.

- **Task 3 review cycle 2 — raw-value capture, robust slot merge, RawBody preservation, nested grouping recursion, comment-aware !important** ✅ (2026-05-04) — second-round hardening on the pre-pass tokenizer.
  - **Raw-value capture (Rec 1)** — new `CssDeclarationRecovery` + `CssStyleRuleRecovery` records. The preprocessor's `ScanForModernDeclarations` walks each rule body token-aware, detects calls to `oklch`/`oklab`/`color-mix`/`light-dark` (case-insensitive, outside strings/comments/parens via `ContainsModernValueFunction`), and records the authored property/value/`!important` triple. The adapter's `AdaptDeclarationsWithRecovery` merges: existing properties get their values overridden with the recovered raw text (closing the silent-corruption gap on `oklch`); recovery-only properties (the `color-mix`/`light-dark` case where AngleSharp produces an empty body) are appended. So a sheet that authored `.a { color: oklch(0.5 0.2 30) }` now adapts to a `CssDeclaration` with `Value.RawText = "oklch(0.5 0.2 30)"` instead of AngleSharp's bogus `rgba(9, 0, 0, 1)`. Tasks 9–10 will resolve the typed value from the preserved text.
  - **Robust slot merge (Recs 4 + 5)** — replaced `CssAngleSharpRuleSlot` / `CssOpaqueAtRuleSlot` with a single unified `CssRuleSlot` carrying `Kind` (`StyleRule` / `AtRule`), `AtKeyword`, `Prelude`, `RawBody`, and `NestedSlots`. Every top-level rule produces exactly one slot, regardless of whether AngleSharp will preserve or drop it. The adapter's `AdaptSlot` tentatively pairs each slot with AngleSharp's next emitted rule; if the rule kinds don't match (because AngleSharp dropped a previously-supported at-rule), the slot demotes to opaque and the AngleSharp cursor doesn't advance. This means future AngleSharp regressions that drop new rule types — beyond today's `@container` / `@layer` — won't reintroduce the ordinal-drift bug. The merge is sequential, single-pass, and never throws. The previous hard-coded `AngleSharpDroppedAtRules` allowlist still exists as a hint for the preprocessor's tracking of "expected drops" but isn't load-bearing for correctness.
  - **`RawBody` preservation (Rec 2)** — new field on `CssAtRule`. The preprocessor stashes the raw text inside `{...}` for at-rules whose body AngleSharp can't decompose (today: `@container`, `@layer` block-form, `@font-face` is also captured raw though AngleSharp handles it cleanly). The adapter copies `RawBody` through to the emitted `CssAtRule` so the cascade resolver in Task 7 can re-parse on demand without re-reading the source stylesheet. Statement-form `@layer name, name2;` has empty `RawBody`. Verified end-to-end: `@container (min-width: 800px) { .a { color: red } .b { color: blue } }` adapts to a `CssAtRule { Name = "container", Prelude = "(min-width: 800px)", RawBody = ".a { color: red } .b { color: blue }" }`.
  - **Nested modern at-rule recovery (Rec 3)** — preprocessor's `WalkRules` recurses through `@media` / `@supports` / `@keyframes` bodies (allowlist via `GroupingAtRules`), producing a per-grouping `NestedSlots` list on the parent slot. The adapter's `AdaptChildRules` walks those nested slots when adapting grouping-rule children, threading the parent walk's `MergeContext` through (with local `AngleIdx` save+restore) so the global style-rule / page / import counters keep counting correctly across nesting. Result: an `@container` nested inside `@media print` now appears in the `@media`'s `ChildRules` as an opaque `CssAtRule` (was silently lost before); modern value functions inside nested style rules also get their raw values restored.
  - **Comment-aware `!important` (Rec 6)** — extracted `StripTrailingImportant` from the adapter into a new dedicated `ImportantParser` helper. The new implementation walks the value tokenizer-aware throughout: `'/* comment */ !important'`, `'!important /* trailing */'`, and even `'! /* split */ important'` are all correctly recognized. False matches inside strings (`content: "!important"`), parens, and comments are unaffected. Same parser is now used both by margin-box declaration parsing in the adapter and by the preprocessor's modern-value capture path so behavior is consistent.
  - **CssAtRule.RawBody field added** — backward-compatible (default empty string). All call sites pass through the slot's `RawBody`. The `CssParserAdapter.AdaptUnknownGroupingRule` and `AdaptUnknownRule` fallbacks pass empty body since they don't have slot context.
  - **Compatibility matrix reclassified (Rec 7)** — `oklch` / `oklab` / `color-mix` / `light-dark` moved from ✅ to 🧪 with notes citing the AngleSharp limitation and the Task 3 raw-text preservation. `@layer` (both block-form and statement-form) and `@container` parsing similarly reclassified to 🧪. Updates dated 2026-05-04 with the review-cycle 2 reference.
  - 22 new unit tests (~14 in `CssPreprocessorTests` covering modern-value capture across `oklch`/`oklab`/`color-mix`/`light-dark`, !important interaction, function-in-string token-awareness, ordinal counting; ~14 in `CssParserAdapterPreprocessTests` covering raw-value merge in various scenarios, RawBody preserve for opaque at-rules, nested modern-at-rule end-to-end, !important + comments in three positions, robust slot mismatch). Some existing tests adjusted: the `Adapt_nested_rule_locations_are_unknown_not_misaligned` test flipped to verify that nested rule locations are now POPULATED (because of nested-grouping recursion) rather than `Unknown`.

- **Task 4 — Source-generated property tables from `properties.json`** ✅ (2026-05-04)
  - **Hand-written supporting types** under `src/NetPdf.Css/Properties/`: `PropertyType` (16-value `byte`-backed enum: Unknown, Color, Length, LengthPercentage, LengthPercentageAuto, Number, Integer, Percentage, Keyword, String, Url, Time, Angle, Resolution, FontFamilyList, FontWeight, Custom — pragmatic taxonomy that grows with new properties); `AppliesTo` (10-value enum: All, BlockOrInlineOrReplaced, Positioned, BlockOnly, InlineOnly, ListItem, TableElements, ReplacedOnly, FlexItems, GridItems); `ComputedValueKind` (Specified, AbsoluteColor, AbsoluteLength, ResolvedNumber, ResolvedKeyword, Custom); `PropertyMeta` record struct holding the per-property data the cascade consumes.
  - **`properties.json`** at `src/NetPdf.Css/properties.json` — the single source of truth. 37 essential CSS properties for v1: `color` / `background-color`, all 4 `border-*-color`/`-style`/`-width` longhands, all 4 `margin-*` and `padding-*` longhands, `font-family`/`font-size`/`font-style`/`font-weight`/`line-height`, `display`/`position`/`top`/`right`/`bottom`/`left`/`width`/`height`, `text-align`/`text-decoration-line`. Each entry carries `name`, PascalCase `id`, `type`, `default`, `inherit` (per spec), `applies_to`, `computed`. Sorted alphabetically for diff stability. Adding a property is: append + rebuild.
  - **`CssPropertyGenerator`** (`src/NetPdf.SourceGen/CssPropertyGenerator.cs`) — `IIncrementalGenerator` that subscribes to `AdditionalTextsProvider`, picks up `properties.json` from the consuming project's `<AdditionalFiles>` list, and emits `PropertyTables.g.cs` containing `PropertyId : ushort` enum (one value per property, 0-indexed so it doubles as a `Table` index), `PropertyMetadata.Table` (`PropertyMeta[]`), `PropertyMetadata.Count`, and `PropertyMetadata.NameToId` (lazily-built `FrozenDictionary<string, PropertyId>` with case-insensitive comparer). The generator validates inputs and reports `NPDFGEN0001`–`NPDFGEN0004` diagnostics for empty file / missing `properties` array / malformed JSON / missing-or-duplicate fields — build breaks rather than silently emitting wrong code.

The schema is small (object → `properties` array → object entries with primitive fields) so the dependency-free path is simpler. Skips `// line` and `/* block */` comments + tolerates trailing commas as a concession to human-edited JSON.

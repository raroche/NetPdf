# NetPdf Diagnostic Codes

Stable, versioned codes emitted by NetPdf when input cannot be rendered identically to a browser print pipeline. Codes are categorized by prefix and never change meaning once published. New conditions get new codes.

Severity levels:
- **Error** — input rejected; no PDF produced when `Features.StrictUnsupportedCss` is set.
- **Warning** — feature degraded or skipped; PDF still produced.
- **Info** — feature took an alternative path (e.g., raster fallback). PDF unchanged in correctness.

---

## HTML-* — HTML parsing & content

| Code | Severity | Meaning |
|---|---|---|
| `HTML-SCRIPT-IGNORED-001` | Warning | A `<script>` element was encountered. NetPdf does not execute JavaScript in v1. The element was removed from the rendering tree. |
| `HTML-MEDIA-UNSUPPORTED-001` | Warning | A `<video>` or `<audio>` element was encountered. Replaced with empty space. |
| `HTML-IFRAME-UNSUPPORTED-001` | Warning | An `<iframe>` element was encountered. Replaced with empty space. |
| `HTML-JAVASCRIPT-URL-IGNORED-001` | Warning | An `href`/`xlink:href` attribute carried a `javascript:` URL. The attribute was removed so the link will not appear in the emitted PDF; the surrounding element and its text content remain. |
| `HTML-CANVAS-IGNORED-001` | Warning | A `<canvas>` element was encountered. NetPdf does not execute the scripted drawing API. Replaced with empty space. |
| `HTML-OBJECT-EMBED-UNSUPPORTED-001` | Warning | `<object>` or `<embed>` encountered; not supported. |
| `HTML-EVENT-HANDLER-IGNORED-001` | Info | An `on*` event-handler attribute (e.g., `onclick`, `onload`) was found on an element and stripped. NetPdf does not execute scripts in v1; the strip is defense-in-depth so PDF/UA emission can't surface attribute values into accessibility metadata. |
| `HTML-DOM-LIMIT-EXCEEDED-001` | Warning | The parsed DOM exceeded one of the per-document DoS caps (element count, nesting depth, attributes per element, attribute value length, or text-node content length). Excess regions are removed; over-long values + text are clamped with `…`. One diagnostic per violation kind. |
| `HTML-STRIP-NOT-STABLE-001` | Warning | The iterative HTML strip pass did not converge after the maximum iteration cap; dangerous content (script / `on*` handler / `javascript:`/`vbscript:`/`data:` URL) may remain in the DOM. Defense against AngleSharp normalization or SVG `<foreignObject>` re-introducing stripped content. The message names the surviving kinds. |
| `HTML-INPUT-TOO-LARGE-001` | Warning | The HTML input length exceeded the per-document character cap; the parse was rejected before AngleSharp materialized the tree. Defends against very large single-string inputs that would otherwise consume parser CPU + heap. |

## CSS-* — CSS parsing & rendering

| Code | Severity | Meaning |
|---|---|---|
| `CSS-PARSE-WARNING-001` | Warning | A CSS rule was malformed and skipped. |
| `CSS-AT-RULE-UNKNOWN-001` | Info | An unrecognized at-rule was preserved in the AST but had no rendering effect. |
| `CSS-PROPERTY-UNKNOWN-001` | Info | A vendor-prefixed or unknown CSS property was ignored. |
| `CSS-VAR-CIRCULAR-001` | Warning | A `var()` chain produced a circular reference; resolved to fallback or `unset`. |
| `CSS-VAR-EXPANSION-LIMIT-001` | Warning | A `var()` substitution exceeded the user-agent's depth or output-length safety limit (non-cyclic but pathological); resolved to fallback or `unset`. Distinct from circular references — see `CSS-VAR-CIRCULAR-001`. |
| `CSS-CALC-INVALID-001` | Warning | A `calc()` / `min()` / `max()` / `clamp()` / `abs()` / `sign()` expression had invalid syntax or a type mismatch (e.g., adding a number to a length, mixing incompatible units in a context that doesn't allow it). The function text is preserved verbatim so layout can react; the property may compute to its initial value. |
| `CSS-CALC-DIV-BY-ZERO-001` | Warning | A `calc()` expression divided by zero. Per CSS Values L4 §10.1, the result is the IEEE-754 NaN value; the property is treated as "invalid at computed value time" — initial value applies. |
| `CSS-PROPERTY-VALUE-INVALID-001` | Warning | A property's value text could not be parsed into the property's typed value per its declared `PropertyType` (e.g., `color: not-a-color`, `width: nonsense`, `display: foo`). The cascade's "invalid at computed value time" rule applies — the property's initial value (or the inherited value for inherited properties) is used. |
| `CSS-CONTAINER-QUERY-UNSUPPORTED-001` | Warning | `@container` rule encountered; condition not evaluated, contained rules skipped. Roadmap v1.4. |
| `CSS-HAS-RENDERING-NOT-IMPLEMENTED-001` | Warning | `:has()` selector parsed but treated as no-match in v1. Roadmap v1.4. |
| `CSS-BACKGROUND-IMAGE-UNSUPPORTED-001` | Warning | A `background-image` form (or one of its variant longhands) is outside the supported set, so it was ignored; any `background-color` still paints. Supported: a single `url(...)` / `none`, a `linear` / `radial` / `conic` gradient (incl. `repeating-*`), and a multi-layer (comma-separated) list of those on a BODY / element box (each layer painted back-to-front). Fires for: an unrecognized value; a multi-layer list with an unparseable layer; a page-margin-box gradient / multi-layer list (margin boxes take a single `url(...)`); a `background-repeat`/`-size`/`-position` value outside the supported set; a gradient `background-size`/`-position` in a non-absolute unit (e.g. `em`) or other unsupported VALUE; and a gradient whose `background-size`/`-repeat` would tile beyond the tile cap (painted once, untiled). Gradient `background-origin`/`-clip` and SUPPORTED `-size`/`-position`/`-repeat` (absolute/percentage sizes, keyword/length positions, repeat / no-repeat / repeat-x / repeat-y / space / round) ARE honored (tiled) and do NOT fire. Once per render. |
| `CSS-SUBGRID-UNSUPPORTED-001` | Warning | `subgrid` value used; treated as `none`. Roadmap v1.3. |
| `CSS-ANCHOR-POSITIONING-UNSUPPORTED-001` | Warning | `anchor()`/`anchor-size()` used; falls back to `auto`. |
| `CSS-POSITION-STICKY-UNSUPPORTED-001` | Warning | `position: sticky` treated as `relative` in v1. |
| `CSS-TRANSFORM-3D-UNSUPPORTED-001` | Warning | A 3D transform was found; the matrix was projected to 2D. |
| `CSS-TRANSFORM-UNSUPPORTED-001` | Warning | A `transform` value could not be parsed into the supported 2D function set (`translate`/`scale`/`rotate`/`skew`/`matrix` + axis variants) — an unknown function, a malformed argument, or an angle in an unsupported unit (`em`/`rem`/`%` translate offsets and `transform-origin` lengths now resolve); the element painted untransformed. Once per render. |
| `CSS-ANIMATION-UNSUPPORTED-001` | Info | `@keyframes`/`animation`/`transition` were ignored. PDF is static. |
| `CSS-FILTER-RASTER-FALLBACK-001` | Info | A CSS `filter` on an `<img>` was applied via the Skia raster fallback (the decoded image runs through the filter chain — color matrices for grayscale/sepia/invert/brightness/contrast/saturate/hue-rotate/opacity, plus blur / drop-shadow — and re-embeds as a raster XObject). Once per render. |
| `CSS-FILTER-ELEMENT-UNSUPPORTED-001` | Warning | A CSS `filter` on a NON-replaced element (a `<div>` / text box, not an `<img>`) was ignored — filtering a general element's rendered subtree needs a Skia subtree renderer NetPdf doesn't have yet, so the element painted UNFILTERED. Filters on images ARE applied. Once per render. |
| `CSS-FILTER-UNSUPPORTED-001` | Warning | A CSS `filter` VALUE on an `<img>` could not be parsed (a `url(#id)` SVG-filter reference, an unknown function, or a bad argument / color); the filter was dropped and the image painted unfiltered. Once per render. |
| `CSS-CLIP-PATH-RASTER-FALLBACK-001` | Warning | A `clip-path: path("…")` string could not be PARSED, so it was not applied (the element painted unclipped). A parseable `path()` is now clipped NATIVELY (SVG path → PDF path + `W n`/`W* n`, curves preserved), as are the basic shapes inset/circle/ellipse/polygon. Once per render. |
| `CSS-CLIP-PATH-SUBTREE-UNSUPPORTED-001` | Warning | A `clip-path` on an element with CHILDREN clipped only the element's own decoration (background / border / image), not its descendant content (a subtree clip needs the Skia subtree renderer NetPdf lacks). Once per render. |
| `CSS-CLIP-PATH-UNSUPPORTED-001` | Warning | A non-`none` `clip-path` value could not be applied because it isn't a supported basic shape — a `url(#clip)` SVG reference, a bare `<geometry-box>` keyword, a font-relative (em/rem) length, or malformed syntax. The element painted UNCLIPPED (never dropping content silently). Once per render. |
| `CSS-MASK-RASTER-FALLBACK-001` | Info | A `mask`/`mask-image` on an `<img>` was applied via the Skia raster fallback — the image's alpha was multiplied by the mask image's alpha and re-embedded as an XObject + `/SMask`. Once per render. |
| `CSS-MASK-ELEMENT-UNSUPPORTED-001` | Warning | A `mask`/`mask-image` on a NON-image element was ignored (masking a general element's subtree needs the Skia subtree renderer NetPdf lacks); the element painted unmasked. Masks on `<img>` ARE applied. Once per render. |
| `CSS-BORDER-IMAGE-UNSUPPORTED-001` | Info | A non-`url()` `border-image-source` (e.g. a gradient) was dropped — only `url()` sources are supported. (Edge tiling `repeat`/`round`/`space` + `border-image-width`/`-outset` are now honored.) Once per render. |
| `CSS-OPACITY-GROUP-APPROXIMATED-001` | Info | An element with `opacity < 1` was composited with a CONSTANT alpha applied per painting operation (`/ca`+`/CA` ExtGState on its decoration/image + folded into its glyph fill), not as an isolated transparency GROUP. Exact for a non-self-overlapping element (the common case); an element whose own background/border/text overlap composites slightly differently than group opacity. Faithful group isolation needs a transparency-group Form XObject. Once per render. |
| `LINK-URI-UNSUPPORTED-001` | Warning | An `<a href>` was not emitted as a clickable `/Link` annotation because its URI scheme is not on the safe allowlist (http / https / mailto); file: / data: / javascript: / custom schemes + unresolved relatives are blocked. The link text still renders. Once per render. |
| `LINK-FRAGMENT-UNRESOLVED-001` | Warning | A same-document link (`<a href="#id">`) pointed at a fragment id that no element in the document declares, so no clickable `/GoTo` jump was emitted. The link text still renders. Once per render. |
| `SVG-PARSE-FAILED-001` | Warning | An SVG image could not be parsed, so it was not rendered. The message distinguishes a **security-guard rejection** — a prohibited `<!DOCTYPE`/`<!ENTITY` or an over-size document (the XXE / entity-expansion "billion laughs" defense: `DtdProcessing.Prohibit` + `MaxCharactersInDocument`) — from a **malformed** (not well-formed XML) document. Replaces the prior silent `null` + a misleading generic image-decode error. |
| `CSS-SVG-UNSUPPORTED-001` | Info | A rendered SVG image used a feature the renderer doesn't support — an EXTERNAL / non-`data:` `<image>` href, a `filter` with an EXTERNAL `feImage` href or a lighting `kernelUnitLength` / an unknown `filterUnits`/`maskUnits` value, a `filter`/`marker`/`clip-path`/`mask`/`textPath` referencing a wrong-kind or unresolved target, an unresolved `url(#…)` paint server (`fill="url(#…)"` → painted transparent, NOT black), an unknown element (e.g. `foreignObject`), or content beyond the depth / element budget. Shapes, paths, text (incl. %/em coords, spacing, per-glyph `rotate` [global index], `dominant-baseline`, `textPath` on any shape, `textLength`/`lengthAdjust` [single + multi-chunk]), gradients, `<pattern>` paint servers, `filter` graphs (`feGaussianBlur`/`feOffset`/`feColorMatrix`/`feDropShadow`/`feFlood`/`feMerge`/`feComposite`/`feBlend`/`feMorphology`/`feComponentTransfer`/`feDisplacementMap`/`feConvolveMatrix`/`feTurbulence` [incl. `stitchTiles`]/`feDiffuseLighting`/`feSpecularLighting` [incl. `currentColor` + obb light positions]/`feImage` (a `data:` raster OR an element `href="#id"` reference)/`feTile` + named `in`/`in2`/`result` routing [incl. gradient/pattern `FillPaint`/`StrokePaint` + `BackgroundImage`] + the filter region + per-primitive SUBREGIONS + `primitiveUnits` [subregions + offset/blur/morphology/displacement/light-position lengths]), `clip-path`/`mask` references (incl. the `maskUnits` region), `marker` references (markers cascade, EXACT curve tangents), full `preserveAspectRatio`, `<use>`/`<symbol>` viewports, `data:` images, stroke dashes, opacity, and nested viewports DO render. Once per SVG image. |
| `CSS-CONIC-GRADIENT-RASTER-001` | Info | A `conic-gradient` / `repeating-conic-gradient` was painted via the Skia sweep-gradient raster fallback (PDF has no native conic shading); placed as an image XObject with an alpha `/SMask`. Once per render. |
| `CSS-CONIC-GRADIENT-UNSUPPORTED-001` | Warning | A `conic-gradient` / `repeating-conic-gradient` could not be rasterized (the sweep bitmap would exceed the 4096 px / 4 Mpx cap) and was SKIPPED — the background-color shows. Distinct from the Info raster-fallback code. Once per render. |
| `CSS-GRADIENT-ALPHA-UNSUPPORTED-001` | Warning | A TRANSLUCENT `linear-gradient` / `radial-gradient` (a stop with alpha < 1) could not be rasterized (the alpha bitmap would exceed the 4096 px / 4 Mpx cap). A native PDF axial/radial shading is DeviceRGB (no alpha), so rather than DROP the transparency by painting an opaque approximation the gradient was SKIPPED — the background-color shows. Opaque linear/radial gradients always stay native shadings and are never affected. Once per render. |
| `CSS-BOXSHADOW-BLUR-RASTER-001` | Info | A blurred box-shadow (outset or inset) was painted via the Skia raster fallback. Once per render. |
| `CSS-BOXSHADOW-UNSUPPORTED-001` | Warning | A box-shadow form not painted exactly was ignored or approximated — an offset/blur/spread in an unresolvable unit (px + absolute units supported; `em`/`rem` not), or a blurred shadow (outset or inset) too large to rasterize (painted sharp, over the 4096 px cap). Outset AND inset shadows are otherwise painted. Other layers still paint. Once per render. |
| `CSS-TEXTSHADOW-UNSUPPORTED-001` | Warning | A text-shadow was ignored — an offset/blur used a unit the parser can't resolve (px + absolute units supported; `em`/`rem`/`%` not), dropping the whole value. A non-zero blur is RENDERED (see `CSS-TEXTSHADOW-BLUR-RASTER-001`), not flagged here. Once per render. |
| `CSS-TEXTSHADOW-BLUR-RASTER-001` | Info | A blurred text-shadow was ROUTED to the Skia raster fallback (PDF has no native Gaussian blur): the run's glyph outlines are rasterized at 2×, Gaussian-blurred with σ = blur/2, and placed as an image XObject with an alpha `/SMask`. Emitted at collection when a blur is present; an over-cap run (or a font Skia can't read) then falls back to a sharp offset, and a fully transparent layer is skipped. Sharp (blur = 0) shadows stay glyph-shows. Once per render. |
| `CSS-WRITING-MODE-SIDEWAYS-UNSUPPORTED-001` | Warning | `sideways-rl`/`sideways-lr` writing modes treated as `vertical-rl`/`vertical-lr`. |
| `CSS-CONTENT-FUNCTION-UNSUPPORTED-001` | Warning | A `content:` value (on `::before`/`::after`/`::marker`) used a function/keyword not yet handled (`counter()` / `counters()` / `url()` / `image()` / `image-set()` / `linear-gradient()` / `open-quote` / `close-quote` / `no-open-quote` / `no-close-quote`). The pseudo-element generates no box. |
| `CSS-ATTR-MULTI-ARG-UNSUPPORTED-001` | Warning | A modern `attr(name type, fallback)` form was rejected — only the bare `attr(name)` form is supported. The pseudo-element generates no box rather than silently dropping the type / fallback args. |
| `CSS-MODERN-COLOR-FUNCTION-UNSUPPORTED-001` | Info | A modern color function (`oklch()` / `oklab()` / `lab()` / `lch()` / `color()` / `color-mix()`) was used in a property value. These are rejected — the cascade's "invalid at computed value time" rule applies (initial / inherited value used). |
| `CSS-PSEUDO-SUPPRESSED-ON-REPLACED-001` | Info | A `::before` / `::after` rule targeted a replaced element (`<img>` / `<video>` / `<canvas>` / `<iframe>` / `<object>` / `<embed>`); per CSS Pseudo L4 §3 the pseudo-element is suppressed because replaced elements are atomic and can't host generated content. The author rule has no effect. |
| `CSS-VAR-EXPANSION-LIMIT-001` | Warning | A `var()` substitution exceeded a depth, output-length, or per-element cumulative-output budget. |
| `CSS-CONTENT-FUNCTION-UNSUPPORTED-001` | Warning | `content` value used a function or keyword the list parser doesn't accept (e.g., `counter()`, `url()`, `open-quote`), or the per-pseudo generated-content output exceeded the 64 KiB cap. |
| `CSS-RULE-LIMIT-EXCEEDED-001` | Warning | A stylesheet exceeded one of the per-stylesheet / per-rule DoS caps: rule count (50 000 per sheet), declarations on a single rule (256), or selector alternatives in one rule's selector list (1 024). On rule-count overflow the cascade stops processing further rules; on per-rule overflow the declaration / alternative list is tail-truncated. |
| `CSS-CASCADE-OVERFLOW-001` | Warning | The cascade exceeded the per-render cumulative-matched-declaration cap (5 000 000). Catches the compound per-rule × per-element explosion that stays under each individual cap but blows the matched-rule table. Subsequent matches are skipped; emitted once per render. |

## PDF-* — PDF emission

| Code | Severity | Meaning |
|---|---|---|
| `PDF-WRITER-WARNING-001` | Warning | The PDF writer produced output but encountered a non-fatal issue (e.g., a fragment was clipped because it exceeded paper size). |
| `PDF-FONT-NOT-EMBEDDABLE-001` | Warning | A font's license bits forbid embedding. PDF still emitted but the font is referenced, not embedded. |
| `PDF-PAGE-LIMIT-EXCEEDED-001` | Warning | A conversion tried to emit more pages than the configured `SecurityPolicy.MaxPages` cap — a policy-driven, tighter-for-untrusted layer above the pipeline's hard 20 000-page emergency backstop. Rendering stops at the cap and drops the remaining content, so a page-amplifying untrusted document is bounded rather than exhausting memory/time. Raise `SecurityPolicy.MaxPages` for a legitimately long trusted document. |
| `PDF-OUTPUT-SIZE-EXCEEDED-001` | Error | The produced PDF exceeded the configured `SecurityPolicy.MaxOutputBytes` cap. Unlike the page cap this ABORTS the conversion (thrown as `HtmlPdfException`) — a truncated PDF is not meaningful. Bounds output amplification (huge image cascades / page explosions) independently of the page count. Default effectively unlimited; `UntrustedHtml` caps at 50 MiB. |
| `PDF-CONTENT-OVERFLOW-TRUNCATED-001` | Warning | Some content does not fit its page's content box and is clipped — e.g. an element wider than the content area (NetPdf paginates the block axis across pages but does not split the inline axis), content at a negative / absolute offset outside the page, or content beyond the page-count safety cap. Block-axis overflow flows onto further pages; only the un-paginatable inline-axis / off-page remainder is clipped. Surfaced rather than dropped silently. Fires at most once per conversion. |

## LAYOUT-* — Layout

| Code | Severity | Meaning |
|---|---|---|
| `LAYOUT-INLINE-SKIPPED-NO-SHAPER-RESOLVER-001` | Warning | A block container with inline-level children was encountered, but the layouter was constructed without an inline shaper resolver. The inline children are skipped (rendered as empty space) so layout can still complete. Production callers wire a shaper resolver when constructing the renderer; the diagnostic exists for test harnesses and tooling that drive the layouter directly. |
| `LAYOUT-INLINE-ATOMIC-NOT-SUPPORTED-001` | Warning | An inline-only block contained an atomic inline descendant (`inline-block` / `inline-flex` / `inline-grid` / `inline-table` / `inline replaced`). The atomic inline is skipped in the resulting line. One diagnostic per inline-only block aggregates the skipped-atomic count. |
| `LAYOUT-INLINE-UNSUPPORTED-001` | Warning | The inline pass for an inline-only block threw `NotSupportedException` for a configuration the inline layouter doesn't yet support (e.g., per-source-TextRun `word-break: keep-all` mismatch — CJK semantics need UAX #24 script detection). The block emits no fragment + the block layouter advances past it (same chain-reset semantics as the no-resolver skip). The thrown exception's message is carried in the diagnostic's message. |
| `LAYOUT-TABLE-FEATURE-UNSUPPORTED-001` | Warning | The table layouter encountered a CSS Tables L3 feature its current algorithm doesn't implement. `colspan` / `rowspan` cell merging (via the 2D occupancy-grid algorithm) and caption layout (`caption-side: top` / `bottom`) are supported, so spans and captions no longer emit this code. Remaining triggers: `rowspan="0"` / `colspan="0"` HTML5 §4.9.11 "remainder of row-group / column-group" semantics (clamped to 1 + diagnostic carries the cell + axis), and the structural-anomaly case where a `<table>` wrapper has no `TableGrid` child (malformed box tree from `BoxBuilder`). The table renders with the feature silently ignored. See `docs/deferrals.md#table-auto-fixed-spans-borders`. |
| `LAYOUT-TABLE-SLOT-BUDGET-EXCEEDED-001` | Warning | The table layouter's cumulative `rowspan × colspan` slot count exceeded the 1,000,000-slot DoS budget. Cells past the budget are capped at `rowspan = colspan = 1`; the table still renders (truncated geometry, not dropped content) but the author's recorded spans were ignored. Defends against hostile HTML where legal attribute values (e.g., `rowspan="65534" colspan="1000"` on multiple cells) would otherwise force unbounded CPU + memory work in the placement pass. |
| `LAYOUT-TABLE-INLINE-OVERFLOW-001` | Warning | Under `table-layout: fixed`, the sum of declared column widths (from `<col>` / `<colgroup>` / first-row cell widths) exceeded the table wrapper's content-inline-size. CSS 2.1 §17.5.2.1 says the table grid grows to fit the declared widths in that case — the table overflows its wrapper in the inline axis. The layouter keeps the declared widths intact (row + caption fragments grow to the column sum) so author intent is preserved; the diagnostic message names the column sum + the wrapper's content-inline-size so authors can tune their declarations. Also fires under `table-layout: auto` when the sum of min-content column widths exceeds the wrapper's content-inline-size (the shrink-to-fit algorithm cannot narrow columns below their min-content widths without splitting words). See `docs/deferrals.md#table-auto-fixed-spans-borders`. |
| `LAYOUT-TABLE-INTRINSIC-MEASUREMENT-BUDGET-EXCEEDED-001` | Warning | Under `table-layout: auto`, the per-table speculative intrinsic-measurement budget was exceeded. Each cell normally runs two speculative nested `BlockLayouter` passes (min-content at 1px + max-content at 1e6 px) — i.e., 2 ops per cell. For very large tables the cumulative work is capped at 10,000 ops; cells beyond the cap fall back to `(minContent=0, maxContent=contentInlineSize)`, producing a degenerate equal-split-like distribution rather than DoS-amplifying the speculative passes. The diagnostic carries the budget, the cell count, and the number of cells that fell back. Defends against hostile HTML with pathologically deep cell content trees in very large tables. See `docs/deferrals.md#table-auto-fixed-spans-borders`. |
| `LAYOUT-MEASURE-NESTING-BUDGET-EXCEEDED-001` | Warning | Speculative content measurement (the intrinsic min/max-content probes that `NestedContentMeasurer` runs for content-sized flex / grid / table boxes) nested deeper than the budget (`NestedContentMeasurer.MaxMeasureNestingDepth` = 96). Each content-sized box can spawn its own probes, so unbounded nesting of content-sized boxes is ~exponential; past the cap the innermost box measures as 0-extent rather than DoS-amplifying. The cap is generous — no real document nests content-sized boxes this deep, so it never fires in practice. A companion to the speculative-measure out-of-flow skip (out-of-flow descendants are not laid out during a pure measure). Defends against hostile HTML with pathologically deep content-sized box trees. |
| `LAYOUT-RECURSION-DEPTH-EXCEEDED-001` | Warning | Block layout recursed past `BlockLayouter.MaxRecursionDepth` (256) — a DoS guard against pathologically deep untrusted HTML that would otherwise `StackOverflow` and halt the process. The DOM parser's own nesting cap (1024) is higher, so a document nested between the two limits reaches layout and trips this guard. Rather than let an untyped `InvalidOperationException` escape `HtmlPdf.Convert`, the render pipeline catches the guard, degrades to a valid PDF built from the content laid out before the cap, and surfaces this diagnostic. Distinct from `LAYOUT-MEASURE-NESTING-BUDGET-EXCEEDED-001` (which degrades a speculative probe to 0-extent without stopping layout). |
| `LAYOUT-TABLE-REWIND-NOT-SUPPORTED-001` | Warning | The break resolver returned `BreakAction.Rewind` at a table row boundary. The `TableLayouter` does NOT register per-row checkpoints (the outer `BlockLayouter` owns the pre-table rewind frontier), so a resolver-named rewind checkpoint inside the table is a contract violation. The layouter falls back to `Continue` + emits this diagnostic so authors / integrators see the dropped rewind. |
| `LAYOUT-TABLE-ROWSPAN-CROSSES-PAGE-001` | Warning | A row break would cut through a cell whose `rowspan>1` origin row commits on the current page but whose span extends past the break. The layouter keeps rowspan cells atomic across pages: it forces the break BEFORE the rowspan origin row (the whole spanning cell stays together on the next page) when at least one row + optional captions have already committed on the current page; otherwise it falls back to the existing forced-overflow path. The diagnostic message names the rowspan cell's row index + span. Spec-strict rowspan distribution across pages (CSS Tables L3 §11) is not yet implemented. |
| `LAYOUT-TABLE-HEADER-FOOTER-OVERSIZED-001` | Warning | The combined `<thead>` + `<tfoot>` stack height (header rows + footer rows) exceeds the fragmentainer's available block-size, leaving no room to repeat the header + footer on every page along with any body row. Per CSS Tables L3 §3.6 / §11 the header + footer repeat at the top + bottom of each page; if they exceed the fragmentainer no body row can fit on a page that also honors the repeat contract. The layouter commits the header + footer once (atomically) on the current page, skips the body to avoid infinite continuation loops, and surfaces this diagnostic so authors can reduce header / footer content or widen the page. The diagnostic message names the combined header+footer height + the fragmentainer block-size. |
| `LAYOUT-MULTICOL-COLUMN-COUNT-CLAMPED-001` | Warning | An author-supplied `column-count` exceeds the layouter's safety cap (= 1000) and is silently clamped. The cap protects against the per-column arithmetic's O(N)-per-child cost on adversarial inputs (a `column-count: 1000000` would otherwise allocate ~1M per-column geometry slots + invoke 1M nested `BlockLayouter` calls per page — a DoS vector). The clamp itself is intentional; the diagnostic surfaces the cap so authors who legitimately hit it (generated CSS, configuration mistakes) know that the rendered output has AT MOST 1000 columns rather than the requested N. The diagnostic message names the requested `column-count` and the clamped value (= 1000). |
| `LAYOUT-MULTICOL-FORCED-OVERFLOW-001` | Warning | A multicol container's in-flow content can't make forward progress through pagination. Multi-page multicol splits cleanly (via `MulticolContinuation`), including deep-nested multicols (where the multicol sits at recursion depth ≥ 2 inside `EmitBlockSubtreeRecursive`, propagated via the chained `BlockContinuation`), so a clean multi-page split is NOT an error. The diagnostic fires only in the narrow forward-progress safety fallback: a `MulticolLayouter` resume page emits zero fragments AND its continuation doesn't advance past the entry index — the single-oversized-child fallback (analog to `TableLayouter`'s single-oversized-row fallback). The diagnostic message names the column count, the per-column block-size, and the resume child index of the unemitted remainder. See `docs/deferrals.md#multicol-balancing-pagination`. |
| `LAYOUT-MULTICOL-NON-FINITE-GEOMETRY-001` | Warning | An arithmetic combination of `column-count` + `column-gap` would produce non-finite per-column inline-axis geometry (e.g., `column-gap: 1e300` with 100 columns drives `totalGap = (N-1) * columnGap` past `double.MaxValue` → `Infinity`). The CSS resolver's NaN / ±Infinity gate catches most pathological inputs at parse time; this code defends against the multiplicative blow-up that can still arise from individually-finite operands. The layouter clamps `column-gap` so `totalGap < containerInlineSize / 2` and continues emission; rendered geometry differs visually from author intent but doesn't NaN-poison downstream pagination math. The diagnostic message names the original column-gap, the column count, the container inline-size, the (non-finite) total gap, and the clamped value. |
| `LAYOUT-FLOAT-BREAK-INSIDE-NESTED-001` | Warning | A float subtree contains nested multicol or table content that broke mid-emission, returning a `LayoutContinuation` from the in-flow recursion. Floats are out-of-flow per CSS 2.2 §9.5; propagating their continuation through the in-flow pagination machinery requires float-tracking continuation machinery that's a documented deferral. The block layouter discards the returned continuation (atomic-fallback behavior: the float's first-page slice is committed; the remainder is truncated rather than continued onto the next page) + emits this diagnostic so authors see the truncation. Fires at most once per page to avoid spam from pages with many such floats. The diagnostic message names the float box-kind. See `docs/deferrals.md#float-continuation-propagation`. |
| `LAYOUT-FLEX-ITEM-CONTENT-TRUNCATED-001` | Warning | A row-nowrap flex item's measured content block extent reached the practical intra-item content-measurement budget cap (`NestedContentMeasurer.EffectivelyUnboundedBlockBudgetPx`). The atomic measure pass does not paginate, so content TALLER than the cap is clipped; this surfaces the truncation so it is not silent. The cap is huge (~10,400 inches), so this is unreachable for any real document — the page-by-page streaming measure (consuming nested continuations) is the documented follow-up; see `docs/deferrals.md`. |
| `LAYOUT-FLEX-WRAP-REVERSE-APPROXIMATED-001` | Warning | A flex container declared `flex-wrap: wrap-reverse`. The layouter ships `wrap` in full (multi-line greedy packing, per-line alignment, sum-of-lines auto cross-size); `wrap-reverse` requires an additional cross-axis line-stacking reversal transform per CSS Flexbox L1 §6.3 ("same as wrap but the cross-start and cross-end directions are swapped") that is not yet implemented. Until then the layouter approximates `wrap-reverse` as `wrap`: items wrap correctly in main-axis DOM order, but the lines stack in the natural cross-axis direction rather than the reversed direction the author requested. Without this diagnostic the wrong rendering would be silent — the CSS declaration parses successfully but behaves like `flex-wrap: wrap`. Fires at most once per `AttemptLayout` invocation to avoid spam from containers with many items. See `docs/deferrals.md#flex-layouter-features`. |
| `LAYOUT-GRID-TRACK-KIND-UNSUPPORTED-001` | Warning | A `grid-template-rows` / `-columns` track entry uses a kind the GridLayouter doesn't yet size: fr / auto / min-content / max-content / minmax / fit-content / repeat. The AST contract parses these forms; only `<length>` (pixel) tracks are laid out. Non-length tracks contribute 0 px to the track sum + this diagnostic surfaces the silent drop. Fires once per `AttemptLayout`. |
| `LAYOUT-GRID-PLACEMENT-APPROXIMATED-001` | Warning | A grid item's declared placement uses a value the GridLayouter doesn't yet support: `span N`, `<custom-ident>` (= named line), `<custom-ident> N`, OR a non-default `grid-{row,column}-end` (= author requested a multi-cell span). Only integer line-number placement + single-cell items are supported; the diagnostic surfaces the author intent being approximated as auto-placement / shrunk to a single cell. Fires per item per axis. |
| `LAYOUT-GRID-IMPLICIT-TRACK-UNSUPPORTED-001` | Warning | A grid item was placed at a cell OUTSIDE the explicit grid bounds (= row/column index exceeds the declared track count, OR a 0-track grid has no cells to place into). Per CSS Grid §7.5 the implicit grid should auto-generate tracks via `grid-auto-rows` / `grid-auto-columns`; implicit tracks are not yet supported → the item silently drops (no fragment emitted). Fires once per dropped item. |
| `LAYOUT-GRID-NON-FINITE-GEOMETRY-001` | Warning | A grid container's cumulative track positions or fragment geometry overflowed to a non-finite value (NaN / ±Infinity). Individual track sizes are validated finite at AST-construction time, but cumulative sums can still overflow when summing very large finite tracks (= hostile CSS like `grid-template-rows: 1e300px 1e300px`). The flex-factor sum is also checked: `1e308fr 1e308fr` overflows `flexFactorSum` to ∞ which would silently collapse tracks to 0; the guard detects this + emits this code instead. The layouter detects the non-finite value + skips item emission so paint / PDF cannot be corrupted by garbage geometry. Mirrors `LAYOUT-MULTICOL-NON-FINITE-GEOMETRY-001`. Fires once per dispatch. |
| `LAYOUT-GRID-ZERO-SIZED-CELL-CONTENT-SKIPPED-001` | Warning | A grid item's cell resolved to a zero-sized area (inline-size = 0 OR block-size = 0) AND the item has child content. The outer item fragment still emits at the zero-sized geometry, but the sub-BlockLayouter dispatch can't run with a non-positive content extent (= `FragmentainerContext`'s positive-size validation), so the inner content is skipped + this diagnostic surfaces the silent drop. A zero-area grid cell is NOT equivalent to `display: none` per CSS — content should overflow or be clipped per the painter's overflow rules. Fires per item. |
| `LAYOUT-GRID-FR-UNDER-INDEFINITE-APPROXIMATED-001` | Warning | A grid container with auto block-size (or auto inline-size for column-flow grids) contains `fr` tracks on the same axis. Per CSS Grid §11.7 flexible tracks under indefinite available space resolve via intrinsic / max-content sizing. The pre-measure can't fold fr contributions into the wrapper's natural extent without that intrinsic branch, so fr tracks on the indefinite axis collapse to 0 + this diagnostic surfaces the approximation. The author's intent (= "fr fills remaining space") doesn't apply when the space is itself indefinite; declare an explicit height/width on the grid container. Fires once per `AttemptLayout`. This is narrowed to fire only when the axis extent is GENUINELY indefinite — an auto-`width` grid nested in a definite containing chain (e.g. `repeat(4,1fr)` columns inside a flex item) has a DEFINITE used INLINE size per CSS Grid §5.2, so its fr columns resolve exactly and no longer emit this (false-positive) warning. The BLOCK axis keeps the slot-based behavior (an auto-height grid's block extent is the fragmentainer fallback, still indefinite). |
| `LAYOUT-GRID-PERCENTAGE-TRACK-APPROXIMATED-001` | Warning | A grid track uses a percentage value at the top level (`grid-template-columns: 50%`) OR inside a `minmax()` / `fit-content()` sub-arg (`minmax(50%, 1fr)`, `fit-content(25%)`). The sizing path doesn't yet resolve percentages against the container's extent; treating them as their nominal pixel value would silently produce wrong layouts. The track contributes 0 px so the author sees a clearly broken layout + this diagnostic instead of an authoritative-looking-but-wrong size. Fires once per `AttemptLayout`. |
| `LAYOUT-GRID-MAX-EXPANDED-TRACKS-TRUNCATED-001` | Warning | A grid track list's `repeat(N, ...)` expansion would exceed `TrackList.MaxExpandedTrackCount` (50,000). Expansion truncates at the cap to prevent unbounded memory allocation from hostile CSS like `repeat(10000, 1px 1fr 1px 1fr 1fr 1px)` (= 60,000 expanded entries). Items placed at indices in the truncated tail are silently dropped (= no fragment emitted). Combined with `TrackRepeat.MaxRepeatCount = 10_000` at parse time, this bounds total CPU + allocation per grid container. Fires once per `AttemptLayout`. |
| `LAYOUT-GRID-AUTO-FIT-APPROXIMATED-001` | Warning | **RETIRED — no longer emitted.** Previously flagged that `repeat(auto-fit, …)` expanded identically to `auto-fill`. The CSS Grid L1 §7.2.3.1 empty-track collapse is now implemented: empty auto-fit tracks collapse to 0 + their gutters merge after placement, so `repeat(auto-fit, …)` compresses for real (and differs from `auto-fill`). The code constant is retained so regression tests can assert it is NOT produced. |
| `LAYOUT-GRID-FORCED-OVERFLOW-001` | Warning | A single grid row's height exceeds the fragmentainer's block-axis budget on its first attempt UNDER `LayoutAttemptStrategy.LastResort` (= the row is the FIRST committed on the current page, but its height alone is larger than the available space, AND the dispatch has exhausted rewind/retry options). Per CSS Fragmentation L3 §4.4 progress rule the row is force-emitted to prevent pagination deadlock; content overflows the fragmentainer-block-end region. Under `Strict` strategy (= rewind-retry available) the layouter defers the entire grid instead of force-emitting. The continuation still proceeds to the next page at row K+1. Row-atomic pagination is supported; intra-row item splitting is post-v1. Fires per offending row. |
| `LAYOUT-GRID-RESUME-INLINE-SIZE-MISMATCH-001` | Warning | A grid resume continuation arrives at a page with a different `contentInlineSize` than the cache was built for (e.g., left/right pages with different margins, or nested fragmentainers). The cached fr / Maximize'd column widths are stale at the new inline size; the cache is invalidated + a fresh §11 sizing + §8.5 placement pass runs. Note that sparse auto-placement is order-sensitive — a different placement may emerge from the fresh resolve if items were partially emitted on the prior page. Callers should avoid resuming a grid into a page with a different inline content size. Fires once per resume invalidation. |
| `LAYOUT-GRID-RESUME-CACHE-REJECTED-001` | Warning | A grid resume cache was rejected by the receiving `GridLayouter` for a structural anomaly: the cache's `GridIdentity` doesn't match the receiving `rootBox` (= cache routed to the wrong grid), array lengths inconsistent, an item placement out of bounds, geometry non-finite, or item payload not a `Box`. The cache is rejected + a fresh resolve runs. Indicates a layouter-dispatch bug in `BlockLayouter` continuation routing. Fires per validation failure. |
| `LAYOUT-ABSOLUTE-FEATURE-UNSUPPORTED-001` | Warning | A `position: absolute` box used a feature the placement doesn't implement, so the box is DROPPED (no fragment emitted) rather than mis-placed. Only EXPLICIT pixel `top`/`left` + `width`/`height` anchored to the establishing block's content box are supported. Triggers: `auto` offsets (static-position resolution), `right`/`bottom` anchoring + over-constrained resolution, percentage `top`/`left`/`width`/`height`, and `auto` width/height (shrink-to-fit / offset-derived sizing). The diagnostic message carries the specific deferred property. See `docs/deferrals.md#abspos-cycle-1-explicit-only`. |

| `PAINT-BORDER-STYLE-APPROXIMATED-001` | Info | A `border-style` / `outline-style` painted as a solid-ring approximation. `dashed` / `dotted` / `double` render faithfully on the square per-edge path, the UNIFORM rounded border ring, and sharp/rounded outlines, so this fires only for: (a) a 3D style (`groove` / `ridge` / `inset` / `outset`) on a ROUNDED ring or outline (per-side bevel can't follow a concentric rounded ring), or (b) a ROUNDED border with NON-UNIFORM (mixed) per-edge non-solid styles (painted via the clipped per-edge path — straight dashes clipped to the outline + square inner corners). `solid` / `none` / `hidden` never emit this (a tracked follow-up, `docs/deferrals.md#layout-to-pdf-pipeline`). Once per conversion. |
| `PAINT-BORDER-ALPHA-APPROXIMATED-001` | Info | **Reserved — no longer emitted.** Was emitted when a partial-alpha border color was painted fully opaque — the border counterpart of `PAINT-BACKGROUND-ALPHA-APPROXIMATED-001`. | `PAINT-TEXT-FONT-UNRESOLVED-001` | Warning | A font could not be used for some text, so that text was not painted. Emitted by the render pipeline when a `font-family` stack resolves to no face (or the bytes are unsafe / WOFF-wrapped) — text shaping runs during layout, so this is caught as a backstop — and by the `TextPainter` when a resolved font can't be parsed / subset / embedded (e.g. a CFF/OTF outline font). The rest of the document still renders; only the affected text is skipped. Surfaced rather than dropped silently. Deduplicated within a conversion. A bundled deterministic last-resort font (so the default path always resolves) is a tracked follow-up — `docs/deferrals.md#layout-to-pdf-pipeline`. |
| `PAINT-BG-IMAGE-TILE-CAP-001` | Warning | **Reserved — no longer emitted.** Was emitted when a `background-image` tiling exceeded the 4096-tile per-fragment cap (a tiny tile over a large box — a content-stream DoS guard) and the image was skipped. Large tilings now emit ONE PDF tiling-pattern fill (ISO 32000-2 §8.7.3 — O(1) content-stream size for any count), so the cap and its skip path became unreachable. Constant retained for code-registry stability. |
| `PAINT-MARGIN-BOX-CONTENT-OVERFLOW-001` | Warning | A page margin box's laid-out content overflows its content area and was CLIPPED. Two variants, both surfaced rather than clipping silently: **taller** — the content block-height exceeds the available height (a vertical edge box at the band limit, an explicit height, or a multi-line `element()` running header); the overflow is clipped at LINE granularity (the first N whole lines that fit paint, the rest are truncated; 0 fit → decoration-only box), the message naming the box + measured vs available height + the kept/total line count. **wider** — a surviving line's unbreakable content exceeds the content-box width (a long word in a narrow band, a clamped rigid sibling, a nowrap line); the protruding GLYPHS are clipped at the box's padding-box edge via a PDF `q <rect> re W n … Q` clip path (glyph operators still emit, the clip is visual). Fired once per overflowing box per axis. An EXPLICIT `overflow: visible` declared on the box opts out — the content spills and this code does NOT fire (authored overflow; note clip-by-default inverts the spec-initial `visible` — a documented approximation, `docs/deferrals.md#layout-to-pdf-pipeline`). |

## Diagnostic structure

Every diagnostic carries:

```csharp
public readonly record struct Diagnostic(
    string Code,             // stable, e.g. "CSS-CONTAINER-QUERY-UNSUPPORTED-001"
    string Message,          // human-readable, may include input context
    DiagnosticSeverity Severity,
    SourceLocation? Location // file/line/col when available, e.g. CSS source position
);
```

Consumed via:

```csharp
var result = HtmlPdf.ConvertDetailed(html);
foreach (var d in result.Warnings)
    Console.WriteLine($"[{d.Code}] {d.Message}");
```

Or streamed live via `HtmlPdfOptions.Diagnostics: IDiagnosticsSink`.
